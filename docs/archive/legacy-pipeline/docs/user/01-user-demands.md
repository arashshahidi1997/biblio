# User Demands and Expectations

This system is designed for researchers who work deeply with literature and need infrastructure that makes their reasoning transparent, document-grounded, and shareable.

## Core user demands

### 1. “I want to know where a claim comes from.”
Users need to trace statements back to exact locations in papers.

### 2. “I want to understand why a paper is cited.”
Not all citations are equal; users care whether a reference supports a method, a result, or background context.

### 3. “I want confidence that I’m not missing key papers.”
Users want structured ways to explore related work beyond keyword search.

### 4. “I want to reuse this knowledge when writing.”
Extracted insights must translate directly into manuscripts, notes, and figures.

### 5. “I want reproducibility.”
Users expect that re-running analyses on the same corpus yields consistent results.

---

## Common pain points

- Duplicate entries for the same paper.
- Confusion between preprints and published versions.
- Citation managers that store PDFs but do not understand them.
- LLM tools that produce fluent but uncited answers.
- Difficulty maintaining a consistent mental map of a large literature.

This system is explicitly designed to address these failures by externalizing the researcher’s mental map into a computable, inspectable bibliography grounded in documents and citations.
