# Paper summary pipeline as biblio derivative

## Overview
- LLM-generated structured summaries stored as `bib/derivatives/summaries/{citekey}.md`
- Fits the existing derivative pattern (like docling, grobid)
- Site builder already scans for `*summary*` files — zero integration cost

## Pipeline
- `paper_context(citekey)` → docling markdown + grobid header/references → LLM prompt → structured summary
- Output sections: contribution, method, key findings, limitations, relevance to project
- `biblio summarize <citekey>` CLI command + `biblio_summarize` MCP tool
- Batch mode: `biblio summarize --status unread` to process all unread papers

## Automation layer
- Deterministic: CLI command that assembles context and calls LLM API
- Could also be a Claude job / worklog task for batch processing
- Summary is a prerequisite for presentation generation, lit reviews, and comparison tables
