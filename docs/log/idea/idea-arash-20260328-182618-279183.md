# Reading list curation and prioritization

## Overview
- Given a research question or paper draft, automatically prioritize the unread queue
- "Given what you're writing about X, read these 5 papers next, in this order, because..."

## Approach
- Input: research question, draft section, or topic description
- Score unread papers by relevance using RAG + concept metadata
- Consider citation graph proximity to already-read papers
- Output: ranked reading list with per-paper justification

## Integration points
- Builds on paper summaries (needs at least abstracts/summaries to score)
- Uses library status to filter to unread/queued papers
- Could update library priority field automatically
- Natural MCP tool: `biblio_reading_list(question, count=5)`
