# Auto-tagging pipeline for ingested papers

## Overview
- After ingest, papers have title + abstract (BibTeX) and optionally full text (Docling) — enough signal to auto-classify
- **Tier 1 — BibTeX keyword extraction**: sync the `keywords` field from BibTeX into library tags at ingest time; zero LLM cost, immediate value
- **Tier 2 — LLM classification**: small prompt on abstract → 3-5 faceted tags from the vocabulary; runs as a post-ingest hook
- **Tier 3 — Reference propagation**: if N cited papers share a tag, propagate it to the citing paper with lower confidence
- Auto-generated tags should be prefixed `auto:` (e.g. `auto:domain:nlp`) so users can distinguish them from human-curated tags

## Motivation
- Manual tagging doesn't scale — a 200-paper library rarely gets tagged consistently
- Abstract-based classification is cheap (one small LLM call per paper) and surprisingly accurate
- Keyword sync from BibTeX is completely free and covers many papers

## Architecture
- **Post-ingest hook** in `ingest.py`: after successful DOI resolution, extract BibTeX keywords → write to library.yml
- **Auto-tag function** in new `autotag.py`: takes citekey, reads abstract, calls LLM with vocabulary context, returns suggested tags
- **Batch mode**: CLI command `biblio library autotag [--all | --untagged]` to backfill existing papers
- **Scheduled job**: worklog schedule to periodically tag `status:unread` papers with no tags

## Notes
- Tier 1 is a prerequisite — get keyword sync working before adding LLM
- LLM call should be optional (skip if no API key configured)
- Consider caching: store auto-tag results so re-runs are idempotent
- Reference propagation (Tier 3) depends on GROBID reference graph being populated
