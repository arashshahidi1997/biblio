# Smart collections via saved query filters

## Overview
- Current collections are **static** — manually curated lists of citekeys
- Propose adding **smart collections** defined by a query/filter expression that dynamically resolve their membership
- Example: `{"name": "Unread transformers", "query": "tag:method:transformer AND status:unread"}` automatically includes any paper matching the filter
- Smart and manual collections coexist — manual for curated reading lists, smart for living views
- UI shows smart collections with a distinct icon; membership is read-only (edit the query instead)

## Motivation
- Manual collection maintenance doesn't scale — papers fall out of date as status changes
- Researchers think in facets ("show me all unread papers on diffusion models") not in folder assignment
- Bridges the gap between tags (per-paper metadata) and collections (grouped views)

## Design
- Extend `collections.json` schema: if a collection has a `query` field, it's smart; if it has `citekeys`, it's manual
- Query language: simple boolean over `tag:`, `status:`, `priority:`, `author:`, `year:` predicates
- Resolution: evaluate query against library.yml + BibTeX metadata at display time
- CLI: `biblio collection create --smart "tag:domain:nlp AND status:unread" "NLP backlog"`
- MCP: extend collection tools to support smart collection creation and query editing
