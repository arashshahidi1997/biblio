# Controlled tag vocabulary with faceted namespaces

## Overview
- Current biblio tags are free-form strings with no structure or validation
- Propose a **faceted namespace convention** using prefixed tags: `domain:`, `method:`, `task:`
- Examples: `domain:nlp`, `method:transformer`, `task:summarization`, `corpus:arxiv`
- Namespaces give structure without rigidity — unnamespaced tags still allowed for ad-hoc use
- A lightweight vocabulary file (e.g. `bib/config/tag_vocab.yml`) could list known namespaces and suggested values, enabling autocomplete in the UI and validation in CI

## Motivation
- Free-form tags drift quickly — "NLP", "nlp", "natural-language-processing" all coexist
- Faceted tags enable powerful multi-axis filtering (all transformer papers in the vision domain)
- A vocabulary file is low-cost to maintain and provides guardrails without enforcement
