# GUI: Citation drafting, literature review, and presentation tools

## Goal

Add writing-assistance tools to the GUI: a citation drafting panel, a literature review workspace, and presentation template controls. These are the highest-level AI-assisted features exposed through the web UI.

## Context

- These are writing/research workflow tools — they don't fit neatly into a single paper's PaperTab
- Best suited as a new top-level "Research" or "Write" tab, or as panels accessible from the toolbar
- Citation drafting: user types a claim, gets a paragraph with `\cite{}` references
- Literature review: user asks a question, gets a synthesis with citations
- Presentation controls already have a subtab in PaperTab (task 4) — this task adds batch/multi-paper presentation and the template preview
- Backend: `POST /api/cite-draft`, `POST /api/actions/review-query`, `POST /api/actions/review-plan`, `POST /api/actions/present`

## Prompt

> Call `project_context()` and `runtime_conventions()` to understand the workspace.
>
> Read `frontend/src/App.jsx` to understand tab management (the `activeTab` state and how tabs are rendered).
> Read `frontend/src/components/SearchTab.jsx` as a model for a tab with input + results display.
> Read `frontend/src/components/ActionsTab.jsx` for action trigger patterns.
> Read `src/biblio/ui.py` for the cite-draft, review, and present API endpoints.
>
> Implement the following:
>
> ### 1. New "Research" tab
> Add a new top-level tab "Research" (alongside Library, Paper, Corpus, Search, Graph, Setup):
> - Tab button in the main tab bar in App.jsx
> - New component: `frontend/src/components/ResearchTab.jsx`
> - The tab has three sections/subtabs: "Cite Draft", "Literature Review", "Reading List"
>
> ### 2. Citation drafting section
> - Large text input (textarea) for the user's claim, section heading, or topic sentence
> - Citation style selector: LaTeX (`\cite{}`) or Pandoc (`[@]`)
> - Max references slider (1-10, default 5)
> - "Draft Paragraph" button → calls `POST /api/cite-draft`
> - Results area: rendered paragraph with highlighted citation keys
> - "Copy to Clipboard" button (copies the raw text with citation commands)
> - "Copy as Markdown" variant
>
> ### 3. Literature review section
> Two modes:
> - **Query mode**: text input for research question → "Synthesize" button → calls `POST /api/actions/review-query` → renders the synthesis with inline citations, poll status for progress
> - **Plan mode**: citekey input (comma-separated seed papers) + optional question → "Plan Review" button → calls `POST /api/actions/review-plan` → renders the structured plan: scope, themes, gaps, expansion directions, estimated additional papers
> - Results rendered as formatted markdown
> - "Save as Note" button that creates a lit-review note via `POST` to a notes endpoint (or copies content for manual save)
>
> ### 4. Presentation template preview
> In the PaperTab "Slides" subtab (from task 4), enhance with:
> - Template preview descriptions: show what each template (journal-club, conference-talk, lab-meeting) includes
> - After generation, show a "Preview" that renders Marp slide separators as distinct cards/sections
> - "Export" dropdown: HTML, PDF, PPTX (calls the backend which uses marp-cli)
>
> ### 5. Batch actions in ActionsTab
> Add to ActionsTab:
> - "Summarize All Unread" button → `POST /api/actions/summarize` with `{status: "unread"}`
> - "Extract All Concepts" button → `POST /api/actions/concepts-extract` with `{all: true}`
> - "Auto-tag All Untagged" button → `POST /api/actions/autotag` with `{all_untagged: true}`
> - Each shows progress via the existing actionState pattern
>
> Follow existing tab patterns for component structure. After all changes, use `datalad_save` MCP tool to commit with message: "Add Research tab with citation drafting, literature review, and batch actions to GUI".
