# Normalize citekeys: background action with progress and enrichment caching

## Goal

Convert the normalize-citekeys preview from a blocking synchronous request to a background action with progress reporting, and persist any OpenAlex/GROBID enrichment results so they are not re-fetched by future explicit OpenAlex resolve or GROBID runs.

## Context

- Current implementation in `src/biblio/ui.py` at `POST /api/actions/normalize-citekeys` (around line 1039) runs synchronously in the request handler
- The enrichment cascade (lines 1079-1092) does per-paper network calls: DOIâ†’OpenAlex, PDFâ†’GROBID header, Titleâ†’CrossRef â€” all blocking, sequential
- This makes preview very slow for libraries with many non-standard citekeys missing author metadata

- When enrichment fetches OpenAlex data or runs GROBID header extraction, those results should be persisted to their standard cache locations so `biblio openalex resolve` and `biblio grobid run` don't redo the same work later:
