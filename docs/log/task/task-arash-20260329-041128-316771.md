# ORCID-based author publication import

## Goal

Given an author's ORCID, pull all their publications from OpenAlex and offer selective import into the library.

## Prompt

> Call `project_context()` and `runtime_conventions()`.
>
> Read `src/biblio/openalex.py` and `src/biblio/openalex/openalex_client.py` to understand the existing OpenAlex integration. Read `src/biblio/graph.py` for how graph expansion discovers papers via OpenAlex. Read `src/biblio/ingest.py` for the ingest pipeline.
>
> Implement:
> 1. **OpenAlex author query** — add to `openalex_client.py` or a new `src/biblio/author_search.py`:
>    - `search_by_orcid(orcid)` — calls OpenAlex API: `GET https://api.openalex.org/authors?filter=orcid:{orcid}` to get the author record
>    - `get_author_works(author_id, per_page=100)` — paginated fetch of all works by the author
>    - Return: list of works with doi, title, year, journal, cited_by_count, open_access status
> 2. **CLI** — `biblio add --orcid 0000-0002-1234-5678 [--since YEAR] [--min-citations N]` — previews publications, optional filters, then ingests selected DOIs
> 3. **MCP tool** — `biblio_author_works(orcid)` returning the publication list
> 4. **API endpoints**:
>    - `POST /api/authors/search-orcid` — body: `{orcid}`, returns author info + publication list
>    - `POST /api/authors/import` — body: `{dois: [...]}`, ingests selected papers
> 5. **GUI** — add an "Import by Author" section in ActionsTab or a modal:
>    - ORCID input field
>    - "Search" button → shows author name + affiliation + h-index
>    - Publication table with: checkbox, title, year, journal, citations, already-in-library indicator
>    - Filter controls: year range, min citations
>    - "Import Selected" button
> 6. **Tests** — mock OpenAlex API responses
>
> Run `make test`. Then use `datalad_save` MCP tool to commit with message: "Add ORCID-based author publication import".
