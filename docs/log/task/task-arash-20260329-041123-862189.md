# Bulk status and tag updates from ExploreTab

## Goal

Add multi-select to ExploreTab paper list and a bulk action toolbar for setting status, priority, and tags on multiple papers at once.

## Prompt

> Call `project_context()` and `runtime_conventions()`.
>
> Read `frontend/src/components/ExploreTab.jsx` (full file) to understand the paper list rendering, existing filtering, and context menu. Read `frontend/src/App.jsx` for state management. Read `src/biblio/ui.py` for `POST /api/library/{citekey}` and the MCP `library_set_bulk` pattern.
>
> Implement:
> 1. **Multi-select** — add checkboxes to each paper row in ExploreTab. Shift-click for range select. "Select all" / "Select none" controls.
> 2. **Bulk action toolbar** — appears when 1+ papers selected, showing:
>    - Status dropdown (unread/reading/processed/archived) → apply to all selected
>    - Priority dropdown (low/normal/high) → apply to all selected
>    - Tag input (add tags to all selected, with the TagInput autocomplete if available)
>    - "Remove tag" input (remove a specific tag from all selected)
>    - "Delete selected" button (with confirmation)
> 3. **Backend endpoint** — add `POST /api/library/bulk` that accepts `{citekeys: [...], status?, priority?, add_tags?, remove_tags?}` and updates all in one call
> 4. After bulk update, refresh the model data
>
> Follow existing CSS patterns. Run `make test`. Then use `datalad_save` MCP tool to commit with message: "Add bulk status and tag updates from ExploreTab".
