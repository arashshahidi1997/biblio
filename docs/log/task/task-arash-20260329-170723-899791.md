# Split paper view with side-by-side and vertical paper tab layout

## Goal

Add VSCode-style split editor support to PaperTab: view two or more papers simultaneously with "Split Right" and "Split Down" options, each pane having its own independent subtab selection.

## Context

- PaperTab currently shows one paper at a time. Multiple papers can be "open" (`openPaperKeys` array in App.jsx) but only one is visible ‚Äî switched via tab bar.
- The user wants to compare papers side-by-side (e.g. PDF of one paper next to Markdown of another, or two PDFs, or a paper's PDF next to its summary).
- VSCode model: each split pane is an independent editor group. Papers can be dragged between groups. Each group has its own tab bar.
- The same paper can be open in two panes with different subtabs (e.g. left pane shows PDF, right pane shows Summary).
- This also enables: reading a paper's PDF while looking at its references, or comparing two papers during a literature review.
