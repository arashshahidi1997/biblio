# RAG build button with confirmation in SearchTab and build status

## Goal

Expose "Build RAG Index" as a prominent button in SearchTab with a confirmation dialog (since it's costly), show index build status/staleness, and display a clear message when the index doesn't exist yet instead of a confusing error.

## Context

- SearchTab currently shows "RAG index not built yet. Run 'Build RAG Index' first." as an error when searching — but there's no build button in SearchTab itself
- "Build RAG Index" exists in ActionsTab but users may not find it there when they're in SearchTab trying to search
- Building the RAG index is expensive (embeds all docling markdown) — needs confirmation
- The index may become stale when new papers are added/docling-processed — should show staleness indicator
- The build is already a background action with status polling (`POST /api/actions/rag-build`, `GET /api/actions/rag-build/status`)
- The RAG index is shared with projio-level indexio — same Chroma DB at `.cache/rag/chroma_db`

## Prompt

> Call `project_context()` and `runtime_conventions()`.
>
> Read `frontend/src/components/SearchTab.jsx` (full file) — understand how semantic search is triggered and how errors are displayed.
> Read `src/biblio/ui.py` — the `/api/search/semantic` endpoint (where the error message comes from, around line 1578-1618), the `/api/actions/rag-build` endpoint, and the `/api/actions/rag-build/status` endpoint.
> Read `frontend/src/components/ActionsTab.jsx` to see the existing "Build RAG Index" button.
>
> Implement:
>
> ### 1. RAG index status endpoint
> Add `GET /api/rag/status` in ui.py that returns:
> - `exists`: boolean — does the Chroma DB directory exist?
> - `last_built`: timestamp of the most recent build (from the DB directory mtime or a metadata file)
> - `docling_count`: number of docling markdown files available
> - `indexed_count`: number of documents in the Chroma collection (if indexio supports this)
> - `stale`: boolean — true if new docling files exist since last build
> - `stale_count`: number of papers with docling but not yet indexed
>
> ### 2. SearchTab — empty state
> When RAG index doesn't exist, instead of an error on search attempt, show a friendly empty state:
> - Icon/illustration area
> - "Semantic search requires a RAG index"
> - "Your library has N papers with text extraction (docling). Build the index to enable semantic search across all papers."
> - Prominent "Build RAG Index" button with the count: "Build Index (N papers)"
> - Confirmation dialog on click: "This will embed N documents and may take several minutes. Proceed?" with [Build] [Cancel]
>
> ### 3. SearchTab — staleness banner
> When index exists but is stale (new docling files since last build):
> - Show a subtle banner above the search input: "Index is outdated — 12 new papers since last build. [Rebuild]"
> - "Last built: 2 hours ago · 195/207 papers indexed"
>
> ### 4. Build progress in SearchTab
> When a build is in progress (triggered from SearchTab or ActionsTab):
> - Show progress inline in SearchTab: "Building index... 45/207 documents embedded"
> - Disable the search input during build
> - On completion, auto-enable search and clear the staleness banner
>
> ### 5. Index stats in SearchTab header
> Show a small stats line below the search input:
> - "195 papers indexed · Last built 2h ago" (when index exists)
> - Clicking the stats opens a tooltip with: index size, embedding model, chunk settings
>
