# PDF annotation extraction as derivative

## Goal

Extract highlights, comments, and annotations from PDF files and store them as a derivative, viewable in the GUI.

## Prompt

> Call `project_context()` and `runtime_conventions()`.
>
> Read `src/biblio/docling.py` to understand the existing derivative pattern (how docling outputs are stored and served). Read `src/biblio/config.py` for `BiblioConfig` and PDF paths. Read `frontend/src/components/PaperTab.jsx` for how derivatives are displayed (summary subtab pattern).
>
> Implement:
> 1. **Annotation extraction module** — create `src/biblio/annotations.py`:
>    - Use `pymupdf` (aka `fitz`) as an optional dependency — it can extract PDF annotations (highlights, text notes, underlines, strikethroughs)
>    - `extract_annotations(pdf_path)` — returns a list of `{page, type, content, rect, color, created_date}` dicts
>    - `save_annotations(citekey, annotations, repo_root)` — saves to `bib/derivatives/annotations/{citekey}.json`
>    - `render_annotations_md(annotations)` — renders annotations as readable markdown grouped by page
>    - Handle missing pymupdf gracefully: skip with warning
> 2. **CLI** — `biblio annotations extract <citekey> [--all]` — extract and save annotations
> 3. **MCP tool** — `biblio_annotations(citekey)`
> 4. **API endpoints**:
>    - `GET /api/papers/{citekey}/annotations` — returns annotations JSON
>    - `POST /api/actions/annotations-extract` — background action to extract for one or all papers
> 5. **GUI** — add "Annotations" subtab in PaperTab (or section in the Notes subtab):
>    - Show extracted highlights/comments grouped by page
>    - Color-coded by annotation type (highlight=yellow, comment=blue, etc.)
>    - Each annotation shows the highlighted text and any comment
>    - "Extract Annotations" button if none exist yet
> 6. **ArtifactBadge** — add badge for annotations presence
> 7. **Tests** — test annotation parsing with a mock PDF (or test the rendering with sample data)
>
> Note: `pymupdf` should be an optional dependency — add it to extras in pyproject.toml under `[annotations]` extra.
>
> Run `make test`. Then use `datalad_save` MCP tool to commit with message: "Add PDF annotation extraction as derivative".
