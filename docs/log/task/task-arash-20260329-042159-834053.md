# Docling progress streaming and AI derivative GUI triggers

## Goal

Fix docling's "Running..." status to show actual progress (subprocess output streaming), and ensure all single-paper AI derivatives (summary, concepts, auto-tag, slides) are triggerable from PaperTab subtabs with inline status display.

## Context

- Docling for a single file runs as a subprocess in a background thread (`src/biblio/ui.py` around line 554-640). The status dict only updates to "Running Docling for {citekey}..." then jumps to "finished" — no intermediate progress from the subprocess output.
- Docling's subprocess writes progress lines to stderr (e.g. page processing updates). These could be captured and streamed to the status dict.
- The GUI tasks from the earlier batch (schedule `c133d5c1`) added Summary and Slides subtabs to PaperTab, but we should verify:
  - Each single-paper derivative (summary, concepts, auto-tag, slides, annotations) is triggerable from within its subtab/panel with a generate button
  - Generation progress is visible inline (not just in ActionsTab)
  - Results appear immediately after generation completes without full page reload
- GROBID has the same "running..." issue — fix both at once

> Implement:
>
> ### 1. Docling progress streaming
> - In the docling-run thread, capture subprocess stderr line-by-line using `subprocess.Popen` with `stderr=PIPE` instead of `subprocess.run`
> - Parse progress lines from docling output (e.g. "Processing page 3/12", "Converting document...")
> - Update the status dict with: `{running: true, message: "Processing page 3/12", progress: {done: 3, total: 12}, logs: "...accumulated stderr..."}`
> - The frontend already polls status — it just needs to display the updated message
> - Apply the same pattern to GROBID if it produces progress output
>
> ### 2. Per-subtab generation triggers in PaperTab
> Ensure each single-paper derivative has a complete generate → progress → display cycle in PaperTab:
>
> **Summary subtab:**
> - "Generate Summary" button when no summary exists
> - Inline progress: "Generating summary..." with spinner
> - On completion, immediately fetch and render the summary markdown
> - "Regenerate" button when summary already exists
>
> **Slides subtab:**
> - Template selector (journal-club, conference-talk, lab-meeting) + "Generate" button
> - Inline progress with spinner
> - On completion, render the slide markdown
>
> **Concepts panel** (sidebar):
> - "Extract Concepts" button if no concepts exist
> - Show spinner during extraction
> - On completion, render concept pills immediately
>
> **Auto-tag** (sidebar, in LibraryPanel):
> - "Auto-tag" button
> - Show spinner, then refresh tags on completion
>
> For each: use a local component state for the generation status (don't rely solely on the global actionState). Poll the specific action's status endpoint. On completion, fetch the new derivative data and update the display without a full model reload.
>
> ### 3. Unified action progress component
> Create a small reusable `ActionProgress` component:
> ```jsx
> <ActionProgress endpoint="/api/actions/summarize" body={{citekey}} onComplete={handleResult} label="Generating summary" />
> ```
> - Handles: trigger POST, poll status, show spinner + message, call onComplete with result
> - Used by all subtab triggers for consistency
>
> ### 4. Docling/GROBID subtab progress
> When docling or grobid is running for the current paper, show the progress inline in the relevant subtab:
> - Markdown subtab: if docling is running, show "Docling: Processing page 3/12..." instead of "No docling data"
> - Refs subtab: if grobid is running, show progress instead of "No GROBID data"
>
> After all changes, run `make test`. Then use `datalad_save` MCP tool to commit with message: "Add docling progress streaming and per-subtab AI derivative triggers".
