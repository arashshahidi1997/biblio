# Fix graph: expanded refs in focused view, persistent green selection, expand-all confirmation

## Goal

Fix three graph visualization bugs: (1) expanded references not showing in focused neighborhood view, (2) selected nodes turning green/teal and staying that color persistently, (3) "Expand Graph" for all papers needs a confirmation dialog.

## Context

The graph is rendered using Cytoscape.js in `frontend/src/components/ExploreTab.jsx`.

### Bug 1: Focused view doesn't show expanded graph
- User selects a paper, switches to "Focused" view, clicks "Expand Graph" for that paper
- Backend responds "Graph expanded: 77 total candidates"
- But the focused neighborhood view still shows the same nodes — the expanded candidates don't appear
- Root cause likely: after graph-expand completes, the graph data in the model isn't refreshed, OR the focused mode filter doesn't include the newly expanded candidates from the graph-candidates API
- The whole point of focused view + expand is to see a paper's references as nodes and be able to ingest them

### Bug 2: Selected nodes turn green and persist
- Clicking a node makes it teal/green (#1a9e8a) and it stays that color even after clicking elsewhere
- Root cause in ExploreTab.jsx around line 243-248: `node[?active]` style applies `background-color: #1a9e8a` based on a data attribute `active`. The `tap` handler (line 282-298) calls `evt.target.select()` which sets Cytoscape's `:selected` pseudo-class
- The `active` data attribute is likely being set but never cleared when another node is clicked or when clicking the canvas
- The `:selected` style (line 251-261) adds blue shadow — that's fine — but the `active` data flag persists

### Bug 3: Expand Graph for All needs confirmation
- "Expand Graph" button in ActionsTab triggers for all papers with no confirmation
- This is a heavy operation (one OpenAlex API call per paper, rate-limited)
- Should show confirmation: "This will query OpenAlex for N papers. This may take several minutes. Proceed?"
