# Citation graph visualization improvements

## Goal

Enhance the GraphInspector with tag/topic cluster coloring, clickable nodes that open papers, and path highlighting between papers.

## Prompt

> Call `project_context()` and `runtime_conventions()`.
>
> Read `frontend/src/components/GraphInspector.jsx` (full file) to understand the Cytoscape.js graph setup, node/edge data structure, and existing color/size options. Read `frontend/src/App.jsx` for how `graphMode`, `colorBy`, `sizeBy` state works and how the graph interacts with paper navigation.
>
> Implement:
> 1. **Color by tag** — add `colorBy: "tag"` option. When selected, show a dropdown of available tag namespaces (domain, method, etc.). Color nodes by their primary tag in that namespace. Use a categorical color palette. Nodes without the tag are gray.
> 2. **Color by status** — add `colorBy: "status"` option. Color by library status: unread=blue, reading=yellow, processed=green, archived=gray.
> 3. **Clickable nodes** — single-click a local node to select it (existing). Double-click to open the paper in PaperTab (`openInPaperTab`). Show a tooltip on hover with title, year, and key tags.
> 4. **Path highlighting** — select two nodes (shift-click), then highlight the shortest citation path between them. Dim non-path nodes/edges. Show path length.
> 5. **Cluster detection** — add a "Clusters" toggle that runs a simple community detection (e.g. connected components or Louvain if cytoscape has it) and colors nodes by cluster. Show cluster count.
> 6. **Legend** — add a small legend panel showing what colors mean for the current colorBy mode.
>
> Follow existing Cytoscape.js patterns in the file. Run `make test`. Then use `datalad_save` MCP tool to commit with message: "Enhance citation graph with cluster coloring, clickable nodes, and path highlighting".
