# Presentation generation from paper context

## Goal

Generate shareable slide decks from paper context using Marp (markdown-based), with figure transclusion from docling outputs. Supports multiple presentation templates (journal club, conference talk, lab meeting).

## Context

- Source idea: `idea-arash-20260328-182616-769030`
- Docling already extracts figures with captions from PDFs
- Marp is the lightest option: markdown with `---` separators, renders to HTML/PDF/PPTX
- Depends on task 4 (summaries) for structured content to pull into slides
- Should support `--prompt-only` mode

## Prompt

> Call `project_context()` and `runtime_conventions()` to understand the workspace.
>
> Read the source idea note referenced in Context above. Read `src/biblio/docling.py` (especially how it extracts figures and stores outputs), `src/biblio/summarize.py` (from task 4), `src/biblio/cli.py`, and `src/biblio/mcp.py`.
>
> Implement the following:
>
> 1. **Presentation module** — Create `src/biblio/present.py` with:
>    - `assemble_slide_context(citekey, root)`: gathers summary, key figures from docling output, BibTeX metadata. Returns structured content suitable for slide generation.
>    - `generate_slides(citekey, root, template="journal-club", prompt_only=False)`: assembles context, then either returns the prompt or calls LLM to generate a Marp markdown deck.
>    - Output stored as `bib/derivatives/slides/{citekey}.md`.
>
> 2. **Slide templates** — Create template prompts for at least three formats stored in `src/biblio/slide_templates/`:
>    - `journal-club.txt`: title slide, motivation, method, key results (with figures), discussion points
>    - `conference-talk.txt`: title, problem statement, approach, results, contributions, future work
>    - `lab-meeting.txt`: title, context/relevance to lab, method summary, key takeaways, questions
>
> 3. **Figure transclusion** — Extract figure paths from docling JSON output. In generated Marp markdown, use relative image references (`![caption](../../derivatives/docling/{citekey}/figures/fig1.png)`) so slides render correctly from the `slides/` directory.
>
> 4. **Marp frontmatter** — Generated slides should include Marp frontmatter: `marp: true`, `theme: default`, `paginate: true`, and paper title/authors in the header.
>
> 5. **CLI command** — `biblio present <citekey> [--template journal-club|conference-talk|lab-meeting] [--prompt-only] [--export html|pdf|pptx]`. The `--export` flag calls `marp-cli` if installed (skip gracefully if not).
>
> 6. **MCP tool** — `biblio_present(citekey, template, prompt_only)`.
>
> 7. **Tests** — Test context assembly, figure path extraction from docling output, Marp frontmatter generation, and template selection.
>
> After all changes, run tests with `make test`. Then use `datalad_save` MCP tool to commit with message: "Add presentation generation with Marp slide templates".

## Acceptance Criteria

- [ ] Figure transclusion from docling outputs with correct relative paths

- [ ] `--prompt-only` mode for non-LLM use
