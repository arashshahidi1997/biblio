# Unpaywall PDF source and institutional access settings

## Goal

Add Unpaywall as a second OA PDF source (alongside OpenAlex), add EZProxy URL rewriting for institutional access, and expose PDF fetch settings in the GUI SetupTab.

## Context

- `src/biblio/pdf_fetch_oa.py` currently only uses OpenAlex OA URLs (`best_oa_location.pdf_url`, `open_access.oa_url`, `primary_location.pdf_url`)
- Unpaywall API is free, just needs an email: `GET https://api.unpaywall.org/v2/{doi}?email=user@example.com` — often finds OA copies that OpenAlex misses (green OA from institutional repositories, PubMed Central, preprint servers)
- EZProxy is used by most universities for off-campus access. LMU Munich pattern: `https://emedien.ub.uni-muenchen.de/login?url=<target_url>`. Rewriting publisher URLs through the proxy gives access to paywalled PDFs when user has VPN/active session.
- The fetch cascade should be: pool check → local PDF exists → OpenAlex OA URL → Unpaywall OA URL → EZProxy-rewritten publisher URL → add to needs-PDF queue
- Config lives in `bib/config/biblio.yml` and user config at `~/.config/biblio/config.yml`
- SetupTab in `frontend/src/components/SetupTab.jsx` has sections for Docling, GROBID, and RAG config — PDF fetch settings should be a new section
- `src/biblio/ui.py` has setup endpoints at `/api/setup` and `/api/setup/*` for config read/write
