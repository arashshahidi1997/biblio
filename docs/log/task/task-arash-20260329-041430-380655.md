# Duplicate paper detection and lint report

## Goal

Detect duplicate papers in the library (shared DOI, near-identical titles, same OpenAlex ID) and present a lint-style report with merge suggestions.

## Prompt

> Call `project_context()` and `runtime_conventions()`.
>
> Read `src/biblio/bibtex.py`, `src/biblio/citekeys.py`, and `src/biblio/library.py` to understand how paper metadata is stored. Read `src/biblio/tag_vocab.py` for the existing lint pattern (`lint_library_tags`).
>
> Implement:
> 1. **Duplicate detection module** — create `src/biblio/dedup.py` with:
>    - `find_duplicates(repo_root)` — scans all BibTeX entries and returns groups of likely duplicates, detected by:
>      - Exact DOI match (different citekeys, same DOI)
>      - Normalized title similarity (case-insensitive, strip punctuation, >90% match)
>      - Same OpenAlex ID (if resolved)
>    - Each group: `{citekeys: [...], reason: "doi"|"title"|"openalex", confidence: float, suggested_keep: str}`
>    - `suggested_keep` picks the citekey with most derivatives/metadata
> 2. **CLI** — `biblio library dedup [--json]` that prints the duplicate report
> 3. **MCP tool** — `biblio_dedup()`
> 4. **API endpoint** — `POST /api/library/dedup` returning the duplicate groups as JSON
> 5. **GUI** — add a "Duplicates" section in ActionsTab (alongside the Tag Health lint). Show duplicate groups with "Merge" button per group. Merging keeps the suggested citekey and removes the other(s), transferring any unique tags/notes.
> 6. **Tests** — test duplicate detection with mock BibTeX data
>
> Run `make test`. Then use `datalad_save` MCP tool to commit with message: "Add duplicate paper detection and lint report".
