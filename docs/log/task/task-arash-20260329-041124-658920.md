# Paper notes editor in PaperTab

## Goal

Add a "Notes" subtab to PaperTab for editing per-paper markdown notes stored at `bib/notes/{citekey}.md`.

## Prompt

> Call `project_context()` and `runtime_conventions()`.
>
> Read `frontend/src/components/PaperTab.jsx` — the `CONTENT_SUBTABS` array and how subtab content is rendered. Read `src/biblio/library.py` for the `notes_path()` function. Read `frontend/src/utils/markdown.js` for markdown rendering.
>
> Implement:
> 1. **Backend endpoints**:
>    - `GET /api/papers/{citekey}/notes` — returns the raw markdown content of `bib/notes/{citekey}.md`, or empty string if not exists
>    - `PUT /api/papers/{citekey}/notes` — saves markdown content to `bib/notes/{citekey}.md`, creates the file/directory if needed
> 2. **Notes subtab** — add "Notes" to `CONTENT_SUBTABS` in PaperTab
>    - Split view: left side is a textarea editor, right side is rendered markdown preview (toggle between edit/preview/split)
>    - Auto-save on blur or after 2 seconds of inactivity (debounced)
>    - Show "Saved" / "Unsaved changes" indicator
>    - If no notes exist yet, show a friendly "Start writing notes..." placeholder
> 3. **Notes badge** — add an `ArtifactBadge` for notes in the paper header showing whether notes exist
> 4. The site builder in `src/biblio/site.py` already scans for `*notes*` — verify it picks these up
>
> Run `make test`. Then use `datalad_save` MCP tool to commit with message: "Add paper notes editor to PaperTab".
