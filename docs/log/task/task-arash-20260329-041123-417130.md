# Duplicate ingestion guard on DOI

## Goal

Prevent re-ingesting papers that already exist in the library by checking DOI/citekey before network calls.

## Prompt

> Call `project_context()` and `runtime_conventions()`.
>
> Read `src/biblio/ingest.py` thoroughly — understand the ingest pipeline, how DOIs are resolved, and where entries are written. Read `src/biblio/citekeys.py` to understand how citekeys are tracked. Read `src/biblio/mcp.py` for `ingest_dois`.
>
> Implement:
> 1. In the ingest pipeline, before making any network call (OpenAlex/CrossRef), check if the DOI already exists in any BibTeX source file or if the resulting citekey is already registered
> 2. Return a clear status: `"skipped_duplicate"` with the existing citekey
> 3. For `biblio ingest --dois`, report: "3 added, 2 skipped (already in library)"
> 4. Add a `--force` flag to re-ingest even if duplicate detected
> 5. For the MCP tool `ingest_dois`, include skip info in the response
> 6. Add tests for duplicate detection
>
> Run `make test`. Then use `datalad_save` MCP tool to commit with message: "Add duplicate ingestion guard on DOI".
