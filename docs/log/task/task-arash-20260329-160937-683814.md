# RAG build button with confirmation in SearchTab and build status

## Goal

Expose "Build RAG Index" as a prominent button in SearchTab with a confirmation dialog (since it's costly), show index build status/staleness, and display a clear message when the index doesn't exist yet instead of a confusing error.

## Context

- SearchTab currently shows "RAG index not built yet. Run 'Build RAG Index' first." as an error when searching — but there's no build button in SearchTab itself
- "Build RAG Index" exists in ActionsTab but users may not find it there when they're in SearchTab trying to search
- Building the RAG index is expensive (embeds all docling markdown) — needs confirmation
- The index may become stale when new papers are added/docling-processed — should show staleness indicator
- The build is already a background action with status polling (`POST /api/actions/rag-build`, `GET /api/actions/rag-build/status`)
- The RAG index is shared with projio-level indexio — same Chroma DB at `.cache/rag/chroma_db`
