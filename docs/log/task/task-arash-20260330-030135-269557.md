# PDF annotation extraction as derivative

## Goal

Extract highlights, comments, and annotations from PDF files and store them as a derivative, viewable in the GUI.

## Prompt

>
> Read `src/biblio/docling.py` to understand the existing derivative pattern (how docling outputs are stored and served). Read `src/biblio/config.py` for `BiblioConfig` and PDF paths. Read `frontend/src/components/PaperTab.jsx` for how derivatives are displayed (summary subtab pattern).
>
> Implement:
> 1. **Annotation extraction module** ‚Äî create `src/biblio/annotations.py`:
>    - Use `pymupdf` (aka `fitz`) as an optional dependency ‚Äî it can extract PDF annotations (highlights, text notes, underlines, strikethroughs)
>    - `extract_annotations(pdf_path)` ‚Äî returns a list of `{page, type, content, rect, color, created_date}` dicts
>    - `save_annotations(citekey, annotations, repo_root)` ‚Äî saves to `bib/derivatives/annotations/{citekey}.json`
>    - `render_annotations_md(annotations)` ‚Äî renders annotations as readable markdown grouped by page
>    - Handle missing pymupdf gracefully: skip with warning
> 2. **CLI** ‚Äî `biblio annotations extract <citekey> [--all]` ‚Äî extract and save annotations
> 3. **MCP tool** ‚Äî `biblio_annotations(citekey)`
> 4. **API endpoints**:
>    - `GET /api/papers/{citekey}/annotations` ‚Äî returns annotations JSON
>    - `POST /api/actions/annotations-extract` ‚Äî background action to extract for one or all papers

> 6. **ArtifactBadge** ‚Äî add badge for annotations presence
> 7. **Tests** ‚Äî test annotation parsing with a mock PDF (or test the rendering with sample data)
>
> Note: `pymupdf` should be an optional dependency ‚Äî add it to extras in pyproject.toml under `[annotations]` extra.
>
> Run `make test`. Then use `datalad_save` MCP tool to commit with message: "Add PDF annotation extraction as derivative".
