# Batch docling, smart action tips in stats bar, and missing bulk actions

## Goal

Add "Run Docling For All" and "Fetch PDFs For All" batch actions, and add a smart stats bar at the bottom of the GUI that shows library coverage and suggests next actions.

## Context

- ActionsTab has "Run GROBID For All" but NOT "Run Docling For All" — docling only has "Run Docling For Selected" (single paper)
- The docling-run backend already handles single citekey. Batch docling needs to iterate over all papers with PDFs but no docling output.
- "Fetch OA PDFs" exists but coverage stats aren't shown
- User wants a persistent stats bar like: `677 papers · 207 pdf · 1 docling · 677 openalex · 0 grobid`
- The stats bar should show smart tips based on gaps: "207 papers have PDFs but no docling output — Run Docling For All" or "470 papers missing PDFs — Fetch OA PDFs"
- The `/api/model` endpoint already has per-paper derivative flags; stats can be computed from this
