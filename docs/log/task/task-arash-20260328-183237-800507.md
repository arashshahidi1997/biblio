# LLM auto-tagging pipeline

## Goal

Implement Tier 2 (LLM classification) and Tier 3 (reference propagation) of the auto-tagging pipeline. Uses the tag vocabulary from task 1 to constrain LLM output and propagate tags through the citation graph.

## Context

- Source idea: Tiers 2-3 of `idea-arash-20260328-182639-622041`
- Tier 1 (BibTeX keyword sync) is implemented in task 1
- LLM classification: small prompt on abstract → 3-5 faceted tags from vocabulary
- Reference propagation: if N cited papers share a tag, propagate to citing paper
- Auto-generated tags should be prefixed `auto:` to distinguish from human-curated
- LLM call must be optional — graceful skip if no API key configured

## Prompt

> Call `project_context()` and `runtime_conventions()` to understand the workspace.
>
> Read the source idea note referenced in Context above. Read `src/biblio/library.py`, `src/biblio/cli.py`, `src/biblio/mcp.py`, `src/biblio/grobid.py` (for reference graph), and the tag vocabulary module from task 1 (`src/biblio/tag_vocab.py`).
>
> Implement the following:
>
> 1. **Auto-tag module** — Create `src/biblio/autotag.py` with:
>    - `autotag_llm(citekey, root)`: reads abstract from BibTeX/OpenAlex, loads tag vocabulary, calls LLM with a constrained classification prompt, returns suggested `auto:namespace:value` tags. Use the Anthropic Python SDK. Skip gracefully if `ANTHROPIC_API_KEY` is not set.
>    - `autotag_propagate(citekey, root)`: reads GROBID references for the paper, checks what tags the cited papers have in library.yml, propagates tags that appear in ≥N cited papers (configurable threshold, default 3) as `auto:` prefixed tags.
>    - `autotag(citekey, root, tiers)`: orchestrator that runs requested tiers in order.
>
> 2. **Prompt design** — The LLM prompt should: present the tag vocabulary as the constrained output space, include the paper's title + abstract, request 3-5 tags with confidence scores, and return structured JSON.
>
> 3. **CLI command** — Add `biblio library autotag <citekey> [--all | --untagged] [--tier llm|propagate|all]` subcommand. Batch mode processes multiple papers with progress reporting.
>
> 4. **MCP tool** — Add `biblio_autotag(citekeys, tiers)` MCP tool.
>
> 5. **Caching** — Store auto-tag results so re-runs are idempotent. Cache in `bib/derivatives/autotag/{citekey}.yml` with timestamp and tier source.
>
> 6. **Tests** — Test LLM prompt construction (mock the API call), reference propagation logic, caching, and CLI batch mode.
>
> After all changes, run tests with `make test`. Then use `datalad_save` MCP tool to commit with message: "Add LLM auto-tagging and reference propagation pipeline".
