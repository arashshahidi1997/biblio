# GUI: Paper summary and auto-tag actions in PaperTab

## Goal

Add a "Summary" subtab to PaperTab that displays generated paper summaries with a generate button, and integrate auto-tag as an action badge in the artifact row. Also add a "Slides" subtab for generated presentations.

## Context

- PaperTab has `CONTENT_SUBTABS = ["PDF", "Markdown", "Figures", "Refs"]` (line 620)
- Each subtab renders content in a `.panel.paper-content-panel` div
- Artifact badges (line 7-22) show derivative status with action buttons (docling, grobid)
- The right sidebar has LibraryPanel, Related local papers, Neighborhood panels
- Backend provides: `GET /api/papers/{citekey}/summary`, `POST /api/actions/summarize`, `GET /api/papers/{citekey}/slides`, `POST /api/actions/present`
- `/api/model` now includes per-paper `has_summary`, `has_concepts`, `has_slides`, `has_autotag` flags
- Summary content is markdown — can use the existing `renderMarkdown` utility
