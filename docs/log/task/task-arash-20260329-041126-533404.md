# Library statistics dashboard panel

## Goal

Add a statistics overview panel to the GUI showing library health: papers by status, year distribution, tag coverage, derivative completeness.

## Prompt

> Call `project_context()` and `runtime_conventions()`.
>
> Read `frontend/src/App.jsx` and `frontend/src/components/ExploreTab.jsx` to understand where stats could fit. Read `src/biblio/ui.py` for the `/api/model` endpoint to see what data is already available.
>
> Implement:
> 1. **Backend** — add `GET /api/stats` endpoint that computes:
>    - Total papers count
>    - Papers by status (unread/reading/processed/archived/unset)
>    - Papers by year (histogram data)
>    - Top 15 tags by frequency
>    - Tag namespace distribution (how many domain:, method:, task: tags)
>    - Derivative coverage: % with PDF, % with docling, % with grobid, % with summary, % with concepts
>    - Papers missing PDF count
>    - Papers in fetch queue count
> 2. **Frontend** — add a stats panel. Options for placement:
>    - A collapsible panel at the top of ExploreTab (preferred — visible when browsing)
>    - Or a dedicated "Dashboard" subtab in the library view
>    - Show: status donut chart (CSS-only, no chart library), year histogram (CSS bar chart), tag cloud, derivative coverage bars
>    - Each stat should be clickable to filter the paper list (e.g. click "unread: 42" filters to unread)
> 3. **CSS** — minimal chart styling using CSS variables, no external chart library needed. Use colored bars/segments.
>
> Run `make test`. Then use `datalad_save` MCP tool to commit with message: "Add library statistics dashboard panel".
