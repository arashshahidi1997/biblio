# GUI: Backend API endpoints for new features

## Goal

Add FastAPI endpoints in `src/biblio/ui.py` for all new backend features so the React frontend can access them. This is the prerequisite for all frontend GUI tasks.

## Context

- `src/biblio/ui.py` is a large FastAPI app with inline component serving
- Existing pattern: `@app.get`/`@app.post` endpoints at `/api/...`
- Background actions use a thread + status polling pattern (see docling-run, grobid-run, openalex-resolve as examples)
- New backend modules to expose: `tag_vocab`, `autotag`, `summarize`, `concepts`, `compare`, `reading_list`, `cite_draft`, `lit_review`, `present`, `query`, updated `collections`
- `/api/model` returns the full data model ‚Äî needs derivative flag extensions
- PaperTab subtabs are `["PDF", "Markdown", "Figures", "Refs"]` ‚Äî new subtabs will need content endpoints
- Collections API at `/api/collections` needs smart collection support
