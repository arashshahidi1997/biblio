# GUI: Tag vocabulary autocomplete and library lint panel

## Goal

Upgrade the tag input in PaperTab's LibraryPanel to support vocabulary-aware autocomplete with faceted namespaces, and add a lint panel to the ActionsTab that shows tag inconsistencies.

## Context

- Current tag input in PaperTab `LibraryPanel` (line ~28-69 of PaperTab.jsx) is a plain text `<input>` with comma-separated tags
- Backend exposes `GET /api/tag-vocab` (vocabulary) and `POST /api/library/lint` (lint report) — from task 1
- Tag vocabulary has namespaces (`domain:`, `method:`, `task:`, `corpus:`) with suggested values
- ExploreTab has `tagFilter` state — should also benefit from autocomplete
- No external autocomplete library needed — a simple dropdown overlay works

> ### 1. Tag autocomplete component
> Create a reusable `TagInput` component (can be inline in PaperTab.jsx or a new file in `frontend/src/components/`):
> - Fetch vocabulary from `GET /api/tag-vocab` on mount (cache it)
> - As user types, show a dropdown overlay with matching namespace:value suggestions
> - Typing `domain:` should show all domain values; typing `meth` should show `method:` namespace and matching values
> - Clicking a suggestion or pressing Tab/Enter inserts the tag
> - Show existing tags as removable pills/chips above the input
> - Support unnamespaced free-form tags too (no enforcement)
> - Style the dropdown to match the app's dark theme (use CSS variables from App.css)
>
> ### 2. Replace tag input in LibraryPanel
> Replace the plain `<input>` tag field in LibraryPanel with the new TagInput component. Wire up save to the existing `updateLibraryEntry` function.
>
> ### 3. Tag filter autocomplete in ExploreTab
> The ExploreTab has a `tagFilter` state for filtering the paper list. Add the same autocomplete behavior to this filter input.
>
> ### 4. Library lint panel in ActionsTab
> Add a "Tag Health" section to ActionsTab:
> - Button: "Lint Tags" that calls `POST /api/library/lint`
> - Display results: list of issues grouped by type (non-vocabulary, duplicates, inconsistent casing)
> - Each issue shows the affected citekey(s) and suggested fix
> - "Apply suggestion" button per issue that calls `POST /api/library/{citekey}` to update tags
>
> ### 5. Auto-tag action button
> Add "Auto-tag Selected" and "Auto-tag All Untagged" buttons to ActionsTab that call `POST /api/actions/autotag` with appropriate body. Show status via the existing `actionState` pattern.
