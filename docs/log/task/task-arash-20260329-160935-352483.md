# Entry type classification, flagging non-papers, and filtered views

## Goal

Classify BibTeX entries by type (article, book, thesis, dataset, software, etc.), flag non-paper entries, and filter them out by default in library views and batch actions so they don't clog paper-oriented workflows.

## Context

- BibTeX sources contain mixed entry types: `@article`, `@inproceedings`, `@book`, `@phdthesis`, `@mastersthesis`, `@misc`, `@techreport`, `@proceedings`, `@incollection`, `@manual`, `@unpublished`, etc.
- Only some are "papers": `@article`, `@inproceedings`, `@conference`, `@incollection`, `@unpublished` (preprints)
- Others are non-paper references: `@book`, `@proceedings` (the volume, not a paper), `@phdthesis`, `@mastersthesis`, `@techreport`, `@misc` (could be datasets, software, web pages), `@manual`
- Batch actions like "Run Docling For All", "Fetch OA PDFs", "Auto-tag All" should only target papers by default
- ExploreTab should default to showing papers, with an option to include non-papers
- The entry type is already in the BibTeX data ‚Äî just needs to be surfaced and used for filtering
- Smart collections query DSL already has `type:` predicate (from the query.py module) ‚Äî this task makes it first-class
