# GUI: Smart collections in CollectionTree

## Goal

Extend CollectionTree to support smart collections (query-based dynamic membership) alongside existing manual collections. Smart collections show a distinct icon, dynamic member counts, and an editable query field.

## Context

- `frontend/src/components/CollectionTree.jsx` renders a hierarchical tree of manual collections
- Each `CollectionNode` shows: name, member count (`col.citekeys?.length`), expand/collapse, rename, delete, create child
- Backend `GET /api/collections` now returns collections with optional `query` field (smart) vs `citekeys` (manual)
- `PATCH /api/collections/{col_id}/query` updates the smart collection query
- `POST /api/collections` accepts optional `query` field for smart collection creation
- Smart collection membership resolves dynamically — the count comes from the backend
- Query DSL: `tag:method:transformer AND status:unread` etc.

> Read `src/biblio/ui.py` to understand the collections API endpoints (search for `/api/collections`).
> Read `src/biblio/query.py` to understand the query DSL syntax.
>
> Implement the following:

> In `CollectionNode`, check if `col.query` exists. If so, render a different icon (e.g. `⚡` or `🔍` instead of `▤`). Add a tooltip showing the query string.
>
> ### 2. Dynamic member count
> Smart collections don't have `citekeys` — they have dynamically resolved membership. The backend's `GET /api/collections` response should include a `resolved_count` field for smart collections. Display this count instead of `col.citekeys?.length`.
>
> ### 3. Create smart collection dialog
> Add a "New Smart Collection" button (alongside the existing "New Collection" button in the CollectionTree header). When clicked, show a small inline form:

> ### 4. Edit query action
> For smart collections, replace the rename-on-double-click with a query editor:
> - Double-click on a smart collection name still allows renaming
> - Add a small "edit query" icon button (e.g. `✎`) next to the delete button
> - Clicking it opens an inline query editor that shows the current query and saves via `PATCH /api/collections/{col_id}/query`
>
> ### 5. Smart collection filtering

>
> ### 6. Convert manual to smart
> Add a context menu option or button on manual collections: "Convert to Smart Collection". This should prompt the user for a query string, then call `PATCH /api/collections/{col_id}` to convert it.
