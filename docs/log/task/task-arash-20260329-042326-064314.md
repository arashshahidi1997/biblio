# Multi-paper workspace for literature review and cross-paper presentations

## Goal

Design and implement a multi-paper workspace view for operations that span multiple papers: literature reviews, cross-paper comparisons, multi-paper presentations, and reading list results. This is a new GUI concept — not a paper subtab (those are single-paper) and not quite a top-level tab (it's transient, spawned from a selection or query).

## Context

- Single-paper derivatives (summary, slides, concepts) fit naturally as PaperTab subtabs — already handled
- Multi-paper operations don't belong in any single PaperTab — they span across papers
- The Research tab (from earlier GUI tasks) has cite-draft and lit review as form-based tools, but the **results** need a better display than just rendered markdown in a panel
- A literature review produces: synthesis text + source papers + gap analysis — needs to show the review alongside the source papers
- A comparison table is generated from N papers — needs to show the table alongside those papers
- A multi-paper presentation draws from N papers — needs to show slide deck alongside source papers
- The reading list output is a ranked list of papers — needs to link back to individual PaperTabs

## Design concept: Workspace panel

A **workspace** is a transient split view that opens when a multi-paper operation completes:
- Left pane: the generated content (review, comparison table, slides)
- Right pane: a mini paper list showing the source papers, each clickable to open in PaperTab
- Header: title, source operation, "Save as Note" button, "Export" button
- Can be opened from: Research tab results, comparison panel results, reading list results, context menu "Open in Workspace"
